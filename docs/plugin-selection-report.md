# Kilo 引擎 - 插件选型报告

**日期**：2026-04-09
**作者**：研究 Agent
**阶段**：阶段 2 - 文档与研究

---

## 概述

本报告对 Kilo 游戏引擎所需的各类插件中的 C# 库进行了全面分析。基于成熟度、.NET 10 支持、许可协议、性能和社区等因素，本报告为每个类别推荐了具体的库。

---

## 1. 音频插件

### 评估方案

#### 1.1 NAudio

**状态**：令人担忧
- **成熟度**：成熟但维护存在隐患
- **.NET 10 支持**：不确定 - 未发现明确支持
- **许可证**：MIT（开源）
- **跨平台**：仅限 Windows
- **性能**：Windows 音频处理表现良好
- **社区**：用户量大但活跃度下降

**主要发现**：
- 社区反馈表明该库"似乎已不再积极维护或开发"（[GitHub Issue #1194](https://github.com/naudio/NAudio/issues/1194)）
- 仅限 Windows 平台，与跨平台目标冲突
- .NET 10 支持路径不明确

**优点**：
- 成熟的代码库，功能丰富的音频特性
- MIT 许可证，无商业限制
- 庞大的用户基础和文档

**缺点**：
- 维护不活跃
- 仅限 Windows
- .NET 10 兼容性不明确
- 缺乏更新可能带来安全隐患

#### 1.2 Silk.NET.Audio

**状态**：推荐替代方案
- **成熟度**：积极开发中
- **.NET 10 支持**：很可能 - 属于现代 .NET 生态
- **许可证**：MIT（开源）
- **跨平台**：是
- **性能**：高 - 现代实现
- **社区**：活跃，属于 Silk.NET 生态

**主要发现**：
- 是专注于底层 .NET 绑定的 Silk.NET 大型项目的一部分
- 为现代 .NET 平台设计
- 活跃的开发社区
- 天然跨平台设计

**优点**：
- 积极的维护和开发
- 跨平台支持
- 现代架构
- 属于统一的 Silk.NET 生态（输入、音频等）
- MIT 许可证

**缺点**：
- 与 NAudio 相比，高级功能可能较少
- 成熟度不如某些替代方案
- 文档可能不够全面

#### 1.3 FMOD

**状态**：商业方案
- **成熟度**：非常成熟，行业标准
- **.NET 10 支持**：是（通过 C# 包装器）
- **许可证**：商业许可，对独立开发者友好
- **跨平台**：是
- **性能**：优秀 - 专业级
- **社区**：庞大，业界广泛采用

**主要发现**：
- 根据开发预算的分级许可：
  - **低于 $600k**：免费或每款游戏 $2,000
  - **$600k - $1.8M**：每款游戏 $6,000
  - **超过 $1.8M**：每款游戏 $18,000
- 最近对年收入低于 $200,000 的独立开发者免费
- 文档完善的 C# API（[FMOD 许可](https://www.fmod.com/licensing)）

**优点**：
- 行业标准，经过验证的可靠记录
- 专业级功能和性能
- 跨平台支持
- 优秀的文档和工具（FMOD Studio）
- 对许多独立开发者免费
- 定期更新和支持

**缺点**：
- 大型项目的商业许可成本
- 对专有软件的外部依赖
- 许可可能较为复杂
- 非开源

### 音频推荐：**Silk.NET.Audio**

**理由**：
1. **与 Kilo 架构一致**：Silk.NET.Audio 与 Silk.NET 输入模块形成互补，构建一致的生态
2. **跨平台**：原生跨平台支持与 Kilo 目标一致
3. **开源**：MIT 许可证提供最大灵活性
4. **现代 .NET 支持**：为现代 .NET 平台设计，可能支持 .NET 10
5. **积极开发**：持续的维护和改进
6. **成本**：免费，无许可限制

**备选方案**：对于需要专业音频功能的项目，可考虑 FMOD，其许可成本可由高级功能所证明。

3. **跨平台**：原生多平台支持
4. **积极开发**：定期更新和改进

## 3. 资源加载插件

- **跨平台**：是
- **性能**：高性能，SIMD 优化
- **社区**：活跃，维护良好（[GitHub](https://github.com/SixLabors/ImageSharp)）

**主要发现**：
- 支持 .NET 6、.NET 7、.NET 8
- 3.0.0+ 版本放弃 .NET Framework 支持，专注于现代 .NET
- 积极开发，定期发布
- 支持 WebP、JPEG、PNG 等格式

**优点**：
- 成熟且功能丰富
- SIMD 优化实现高性能
- 积极维护
- 跨平台
- 现代 API 设计

**缺点**：
- .NET 10 支持尚未确认（需关注更新）

- **社区**：有限

**主要发现**：

- 最小依赖
- 近期活动或文档有限

**优点**：
- 轻量且快速
- 单头文件 C 库，开销最小
- 公共领域许可证（stb_image）

**缺点**：
- 维护状态不确定
- 文档有限
- 功能少于 ImageSharp
- .NET 10 支持不明确

#### 3.2 3D 模型加载

**状态**：不推荐
- **成熟度**：成熟但已被取代
- **.NET 10 支持**：不明确
- **许可证**：MIT（开源）
- **跨平台**：是（通过原生 Assimp）
- **性能**：良好（原生 Assimp 绑定）
- **社区**：活跃度下降

**主要发现**：
- 原始库维护不够积极
- 已被现代分支取代

##### AssimpNetter

**状态**：推荐用于 3D 模型加载
- **成熟度**：活跃的分支
- **.NET 10 支持**：当前版本目标 .NET 7；很可能支持 .NET 10
- **许可证**：MIT（开源）
- **跨平台**：是
- **性能**：优秀（已更新至 Assimp 5.4.3/6.0.2）

**主要发现**：
- 最新版本：6.0.2.1（2025 年 6 月）
- 目标 .NET 7
- 已添加 ARM64 支持

**理由 - ImageSharp**：
1. **成熟度和功能**：全面的图像处理能力
2. **性能**：SIMD 优化实现高性能
3. **积极维护**：定期更新和改进

5. **跨平台**：原生多平台支持
6. **社区**：庞大且活跃的用户基础

**理由 - AssimpNetter**：
1. **积极开发**：定期更新，使用最新 Assimp 版本
2. **现代 .NET**：目标 .NET 7，有明确的 .NET 10 升级路径
3. **ARM64 支持**：对现代平台很重要
4. **全面的格式支持**：利用 Assimp 丰富的格式支持
5. **MIT 许可证**：无许可限制
6. **社区集成**：在 MonoGame 和其他框架中使用

## 架构集成

所选插件为 Kilo 构建了一致的架构：

```
Kilo 引擎核心
├── 渲染：Veldrid（使用 System.Numerics）
├── 物理：BepuPhysics（使用 System.Numerics）
├── 音频：Silk.NET.Audio
├── 输入：Silk.NET.Input
├── 资源：
│   ├── 图像：SixLabors.ImageSharp
│   └── 3D 模型：AssimpNetter
├── 日志：Serilog
├── 数学：System.Numerics（内置）
└── 序列化：
    ├── JSON：System.Text.Json（内置）
    └── YAML：YamlDotNet
```

**关键优势**：
- **最少的外部依赖**：仅 5 个外部库（Silk.NET、ImageSharp、AssimpNetter、Serilog、YamlDotNet）
- **利用内置功能**：System.Numerics 和 System.Text.Json 是 .NET 的一部分
- **跨平台**：所有选定的库都支持跨平台部署
- **积极维护**：所有选定的库都在积极开发
- **许可证兼容**：全部使用宽松的开源许可证（MIT 或 Apache-2.0）
- **生态一致性**：Silk.NET 以一致的 API 同时提供音频和输入
